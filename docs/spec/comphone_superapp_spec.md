# เอกสารข้อกำหนดระบบ (System Specification)
## โครงการ COMPHONE SUPER APP V5.5

**ข้อมูลธุรกิจ**
*   **ชื่อองค์กร:** ห้างหุ้นส่วนจำกัด คอมโฟนแอนด์อิเลคโทรนิคส์

*   **ประเภทธุรกิจ:** จำหน่ายคอมพิวเตอร์ โทรศัพท์มือถือ อุปกรณ์ต่อพ่วง อุปกรณ์สำนักงาน รับซ่อมอุปกรณ์ไอที ติดตั้งกล้องวงจรปิด (CCTV) รับงานจัดซื้อจัดจ้างจากหน่วยงานราชการ
*   **ระบบเดิม:** Google Apps Script + Google Sheets เป็นฐานข้อมูล มี LINE/Telegram integration

## 1. บทสรุปผู้บริหาร (Executive Summary)

โครงการ COMPHONE SUPER APP V5.5 เป็นการยกระดับระบบปฏิบัติการภายในของ ห้างหุ้นส่วนจำกัด คอมโฟนแอนด์อิเลคโทรนิคส์ โดยต่อยอดจากโครงสร้างพื้นฐานเดิมที่ใช้ Google Apps Script และ Google Sheets ร่วมกับ LINE/Telegram เพื่อก้าวสู่การเป็น "Super App" สำหรับธุรกิจไอทีและงานรับเหมาติดตั้งอย่างเต็มรูปแบบ เป้าหมายหลักคือการลดภาระงานเอกสารของช่างหน้างาน (Zero-Data Entry) เพิ่มความแม่นยำในการจัดการสต๊อกสินค้า เร่งความเร็วในกระบวนการขายและการเงิน และสร้างระบบลูกค้าสัมพันธ์ที่ทำงานอัตโนมัติ

ระบบใหม่นี้จะบูรณาการเทคโนโลยี AI, Automation และ Location-based Services เข้าด้วยกัน แบ่งการทำงานออกเป็น 4 กลุ่มหลัก ได้แก่ กลุ่มฟีเจอร์หน้างานและการติดตั้ง กลุ่มฟีเจอร์สต๊อกและเครื่องมือ กลุ่มฟีเจอร์การเงินและการขาย และกลุ่มฟีเจอร์การตลาดและลูกค้าสัมพันธ์ ซึ่งจะช่วยลดข้อผิดพลาดในการทำงาน เพิ่มยอดขายจากฐานลูกค้าเดิม และสร้างมาตรฐานการให้บริการระดับองค์กร

## 2. ภาพรวมระบบและสถาปัตยกรรม (System Overview & Architecture)

สถาปัตยกรรมของ COMPHONE SUPER APP V5.5 ถูกออกแบบมาเพื่อความคล่องตัว (Agility) และความคุ้มค่า (Cost-effectiveness) โดยยังคงใช้ประโยชน์จาก Google Workspace ecosystem ที่มีอยู่เดิม แต่เสริมด้วยบริการ API ภายนอกและ AI เพื่อเพิ่มขีดความสามารถ

### 2.1 สถาปัตยกรรมหลัก (Core Architecture)
*   **Frontend / User Interface:** 
    *   **LINE Official Account (LINE OA) & LINE Messaging API:** เป็นช่องทางหลักสำหรับช่างในการส่งรูปภาพ แจ้งสถานะ และรับการแจ้งเตือน
    *   **LINE LIFF (LINE Front-end Framework):** สำหรับหน้าจอ UI ที่ซับซ้อน เช่น ดาชบอร์ดสรุปงาน แผนที่ลูกค้า หรือหน้าจออนุมัติเอกสาร
*   **Backend & Logic:** 
    *   **Google Apps Script (GAS):** ทำหน้าที่เป็น Webhook receiver, API router และ Business logic processor
*   **Database & Storage:**
    *   **Google Sheets:** ทำหน้าที่เป็นฐานข้อมูลหลัก (Relational-like Database) เก็บข้อมูลลูกค้า งาน สต๊อก และการเงิน
    *   **Google Drive:** ทำหน้าที่เป็น File Storage เก็บรูปภาพหน้างาน สลิปโอนเงิน และเอกสาร PDF
*   **AI & External APIs:**
    *   **OpenAI API / Gemini API:** สำหรับการประมวลผล Voice-to-Text, การวิเคราะห์รูปภาพ (Image Recognition), และการสร้างเนื้อหาการตลาด
    *   **PromptPay & Bank APIs:** สำหรับการสร้าง QR Code รับเงินและตรวจสอบสลิปโอนเงินอัตโนมัติ (Slip Verification)
    *   **Google Maps API:** สำหรับการทำ Geofencing และ Heatmap

### 2.2 Data Flow
1.  ช่างปฏิบัติงานส่งข้อมูลผ่าน LINE (ข้อความ, รูปภาพ, เสียง, Location)
2.  LINE Messaging API ส่ง Webhook ไปยัง Google Apps Script
3.  GAS ประมวลผลข้อมูล (เรียกใช้ AI API หากจำเป็น เช่น ถอดเสียง หรืออ่านรูปภาพ)
4.  GAS บันทึกข้อมูลลง Google Sheets และจัดเก็บไฟล์ลง Google Drive
5.  GAS สร้างเอกสาร (PDF) หรือข้อความแจ้งเตือน ส่งกลับไปยัง LINE ของช่าง หรือลูกค้า

## 3. รายละเอียดฟีเจอร์หลัก (Core Features)

### 🛠️ 3.1 กลุ่มฟีเจอร์หน้างานและการติดตั้ง (Field Operations)
**เป้าหมาย:** ช่างทำงานง่ายที่สุด ไม่ต้องพิมพ์เยอะ งานไม่ตกหล่น

#### 3.1.1 ระบบสแกน QR Code 12 สถานะ (Job State Machine)
*   **คำอธิบาย:** ระบบจัดการสถานะงานผ่านการสแกน QR Code ที่พิมพ์ติดไว้บนใบงานหรืออุปกรณ์ ช่างเพียงแค่สแกนเพื่อเปลี่ยนสถานะ เช่น "เริ่มเดินทาง", "ถึงหน้างาน", "กำลังดำเนินการ", "รออะไหล่", "ปิดงาน" ระบบจะบันทึกเวลาและอัปเดตสถานะใน Google Sheets ทันที
*   **User Story:** ในฐานะช่างติดตั้ง ฉันต้องการเปลี่ยนสถานะงานโดยไม่ต้องพิมพ์ข้อความ เพื่อให้ทำงานได้รวดเร็วและไม่เสียเวลา
*   **Flow การทำงาน:**
    1. แอดมินสร้างใบงาน ระบบสร้าง QR Code ประจำงานนั้นๆ
    2. ช่างสแกน QR Code ผ่าน LINE
    3. LINE LIFF เปิดหน้าจอให้เลือกสถานะที่ต้องการเปลี่ยน (หรือเปลี่ยนให้อัตโนมัติตาม Logic)
    4. GAS อัปเดตสถานะและ Timestamp ลง Google Sheets
*   **เทคโนโลยี:** LINE LIFF Scanner, Google Apps Script, Google Sheets
*   **ข้อมูลที่ต้องเก็บ:** Job ID, Timestamp, User ID (ช่าง), Status (1-12), Location (Optional)

#### 3.1.2 ระบบคัดแยกรูปภาพอัจฉริยะและ Vision Agent (Smart Photo Handling & Vision Agent)
*   **คำอธิบาย:** ระบบจัดการรูปภาพหน้างานแบบครบวงจร ตั้งแต่การรับภาพจากหลายช่องทาง การจัดเก็บ การใช้ AI วิเคราะห์ความเรียบร้อย ไปจนถึงการสร้างคอนเทนต์การตลาดอัตโนมัติ
*   **User Story:** ในฐานะแอดมินและเจ้าของร้าน ฉันต้องการให้รูปภาพถูกจัดเก็บอย่างเป็นระเบียบ AI ช่วยตรวจความเรียบร้อยของงาน และสามารถนำรูปไปทำการตลาดได้ทันที
*   **รายละเอียดเชิงลึก 4 ขั้นตอน:**

| ขั้นตอน | รายละเอียด |
| :--- | :--- |
| **📥 1. ช่องทางการรับภาพ (Omnichannel Ingestion)** | รองรับการส่งภาพจาก 3 ช่องทางหลัก ซึ่งทั้งหมดจะถูกส่งเข้า OpenClaw เป็นระบบจัดการกลาง:<br>• **ผ่าน LINE Group (ห้องช่าง):** ช่างถ่ายรูปแล้วส่งลงกลุ่ม LINE ระบบจะดักจับรูปทันที<br>• **ผ่าน Telegram Bot (@Comphone101Bot):** ถ่ายรูปส่งให้บอทโดยตรง ซึ่งรองรับการดึงไฟล์ความละเอียดสูงได้ดีกว่า LINE<br>• **ผ่าน Link (URL):** แปะ Link จาก Google Photos, เว็บฝากไฟล์, หรือกล้องหน้างาน ระบบจะดาวน์โหลดมาวิเคราะห์เอง |
| **🗂️ 2. ระบบจัดเก็บอัจฉริยะ (Auto-Routing & Drive Sorting)** | จัดการไฟล์อัตโนมัติเพื่อความเป็นระเบียบ:<br>• **แยกโฟลเดอร์ตามงานอัตโนมัติ:** เก็บรูปใน Google Drive ตามโฟลเดอร์ลูกค้าและ Job ID (เช่น `Drive / รหัสงาน 001-คุณฮอกกี้ / 01_ก่อนทำ` และ `02_หลังทำ`)<br>• **ลดขยะในห้องแชท (Zero Spam):** ระบบจะส่งข้อความสรุปแจ้งเตือนสั้นๆ แทนการส่งรูปซ้ำซ้อนกลับเข้ากลุ่ม |
| **👁️ 3. การวิเคราะห์ด้วย AI (Vision Field Agent)** | ใช้ AI (Vision) ช่วยตรวจสอบและควบคุมคุณภาพงาน:<br>• **แยกประเภท Before/After เอง:** AI วิเคราะห์เนื้อหาในรูปว่าเป็นก่อนหรือหลังติดตั้ง แล้วย้ายเข้าโฟลเดอร์ที่ถูกต้อง<br>• **QC งานเบื้องต้น:** AI ตรวจความเรียบร้อยของงาน (เช่น สายไฟเก็บเรียบร้อยไหม มุมกล้องเหมาะสมไหม มีขยะทิ้งไว้หน้างานหรือไม่) หากพบความไม่เรียบร้อย บอทจะแจ้งเตือนช่างทันที<br>• **Geofencing:** ดึงข้อมูล GPS/EXIF Data จากรูปภาพเพื่อยืนยันตำแหน่งช่าง ป้องกันการเช็คอินหลอก |
| **🚀 4. ต่อยอดการตลาด (AI Marketing Content)** | สร้างคอนเทนต์อัตโนมัติเมื่อจบงาน:<br>• เมื่อมีรูป After เข้ามาและสถานะงานเป็น "ปิดงาน"<br>• AI จะคัดเลือกรูปที่มีมุมสวยและแสงดี<br>• นำรูปมาจัดวางเป็น Collage และปรับสเกลให้เหมาะสม<br>• เขียนแคปชั่นรีวิวอัตโนมัติ (โดยระบุแบรนด์สินค้า เช่น Hikvision, Dahua)<br>• ส่งร่างโพสต์ให้เจ้าของตรวจผ่านแชท หากพิมพ์ "ตกลง" ระบบจะโพสต์ลง Facebook ของร้านทันที |

*   **เทคโนโลยี:** LINE Messaging API, Telegram Bot API, Google Drive API, LLM (Vision), OpenClaw
*   **ข้อมูลที่ต้องเก็บ:** File ID, Drive Folder URL, Job ID, Image Metadata, GPS Coordinates, AI Analysis Results, Generated Caption

#### 3.1.3 ดึงพิกัดอัตโนมัติ (Geofencing Alert)
*(รวมอยู่ในหัวข้อ 3.1.2 ในส่วนของการวิเคราะห์ด้วย AI)*

#### 3.1.4 สั่งงานด้วยเสียง (Voice-to-Workflow)
*   **คำอธิบาย:** ช่างสามารถส่งข้อความเสียง (Voice Message) อธิบายปัญหาหรือสรุปการทำงานเข้า LINE จากนั้น AI จะแปลงเสียงเป็นข้อความ (Speech-to-Text) และสรุปประเด็นสำคัญบันทึกลงระบบเป็น Log
*   **User Story:** ในฐานะช่างที่มือเปื้อนหรือกำลังขับรถ ฉันต้องการสรุปงานด้วยเสียง เพื่อลดความผิดพลาดจากการพิมพ์
*   **Flow การทำงาน:**
    1. ช่างอัดเสียงส่งเข้า LINE
    2. GAS รับไฟล์เสียง ส่งไปให้ OpenAI Whisper API ถอดความ
    3. ส่งข้อความที่ถอดได้ให้ LLM สรุปเป็นหัวข้อ (เช่น ปัญหาที่พบ, อะไหล่ที่ใช้)
    4. บันทึกข้อความสรุปลง Google Sheets ในช่อง Log ของงานนั้น
*   **เทคโนโลยี:** OpenAI Audio API (Whisper), LLM (GPT-4o-mini), LINE Messaging API
*   **ข้อมูลที่ต้องเก็บ:** Audio File URL, Raw Text, Summarized Log

#### 3.1.5 แจ้งเตือนลืมเครื่องมือ (Smart Tool Tags)
*   **คำอธิบาย:** ระบบป้องกันการลืมเครื่องมือ โดยเมื่อช่างกด "ปิดงาน" ระบบจะส่งข้อความแจ้งเตือนให้ช่างตรวจสอบเครื่องมือสำคัญ (เช่น สว่าน เครื่องวัดสายแลน) ทันที
*   **User Story:** ในฐานะช่าง ฉันต้องการให้ระบบเตือนความจำเรื่องเครื่องมือ เพื่อลดปัญหาเครื่องมือหายที่บ้านลูกค้า
*   **Flow การทำงาน:**
    1. ช่างสแกน QR Code หรือกดปุ่ม "ปิดงาน"
    2. GAS ตรวจสอบสถานะ หากเป็น "ปิดงาน" จะหน่วงเวลา 1 นาที แล้วส่ง Push Message เข้า LINE ช่าง
    3. ข้อความแสดง Checklist เครื่องมือที่เบิกไปสำหรับงานนั้น
*   **เทคโนโลยี:** Google Apps Script (Time-driven triggers / ScriptApp), LINE Push Message
*   **ข้อมูลที่ต้องเก็บ:** Tool Checklist per Job

### 📦 3.2 กลุ่มฟีเจอร์สต๊อกและเครื่องมือ (Inventory & Asset Management)
**เป้าหมาย:** ของไม่หาย รู้ล่วงหน้าว่าอะไรจะหมด

#### 3.2.1 ระบบสต๊อก 3 ชั้น (3-Layer Inventory)
*   **คำอธิบาย:** ระบบติดตามสินค้าและอะไหล่ใน 3 ระดับ: 1. คลังหลัก (Main Warehouse) 2. สต๊อกบนรถช่าง (Van Stock) 3. เครื่องมือส่วนตัวช่าง (Personal Tools) ทำให้รู้ว่าของอยู่ที่ใครและรถคันไหน
*   **User Story:** ในฐานะผู้จัดการคลัง ฉันต้องการรู้ว่าสายแลนเหลืออยู่ที่ร้านเท่าไหร่ และอยู่บนรถช่างคันไหนบ้าง เพื่อบริหารจัดการได้ถูกต้อง
*   **Flow การทำงาน:**
    1. ออกแบบ Google Sheets ให้มีคอลัมน์ Location (Main, Van A, Van B, ช่าง ก)
    2. เมื่อมีการเบิกของ แอดมินบันทึกการโอนย้าย (Transfer) จาก Main -> Van A
    3. เมื่อช่างใช้ของหน้างาน ระบบตัดสต๊อกจาก Van A
*   **เทคโนโลยี:** Google Sheets (Relational Database Design), LINE LIFF (สำหรับหน้าเบิกของ)
*   **ข้อมูลที่ต้องเก็บ:** Item ID, Item Name, Quantity, Location, Transfer History

#### 3.2.2 ระบบพยากรณ์ของขาด (Predictive Stocking)
*   **คำอธิบาย:** AI วิเคราะห์ใบงานล่วงหน้า 3 วัน หากพบว่าอุปกรณ์ที่ต้องใช้ (เช่น กล้องวงจรปิดรุ่นเฉพาะ, สายแลน) มีไม่พอในสต๊อกรถช่าง ระบบจะแจ้งเตือนให้เบิกของล่วงหน้า
*   **User Story:** ในฐานะช่าง ฉันต้องการให้ระบบเตือนเตรียมของก่อนออกไปหน้างาน เพื่อจะได้ไม่ต้องวิ่งกลับมาเอาของที่ร้าน
*   **Flow การทำงาน:**
    1. GAS รัน Trigger ทุกคืน ตรวจสอบคิวงานล่วงหน้า 3 วัน
    2. ดึงรายการ BOM (Bill of Materials) ของงานเหล่านั้นมาเปรียบเทียบกับ Van Stock
    3. หากพบว่าของไม่พอ ส่ง LINE Notify ไปยังกลุ่มช่างและคลังสินค้า
*   **เทคโนโลยี:** Google Apps Script (Time-driven triggers), Data Aggregation logic
*   **ข้อมูลที่ต้องเก็บ:** BOM per Job, Van Stock Level, Job Schedule

#### 3.2.3 ตรวจเช็คเครื่องมือรายสัปดาห์ (Weekly Tool Audit)
*   **คำอธิบาย:** ระบบส่งข้อความอัตโนมัติหาช่างทุกคนทุกเย็นวันอาทิตย์ เพื่อให้ถ่ายรูปเครื่องมือหรือติ๊ก Checklist ยืนยันว่าเครื่องมือส่วนตัวและบนรถยังอยู่ครบ
*   **User Story:** ในฐานะเจ้าของร้าน ฉันต้องการให้ช่างรับผิดชอบเครื่องมือของตนเอง เพื่อลดต้นทุนการซื้อเครื่องมือใหม่
*   **Flow การทำงาน:**
    1. GAS รัน Trigger ทุกวันอาทิตย์ 17:00 น.
    2. ส่ง Flex Message เข้า LINE ช่างแต่ละคน พร้อมปุ่ม "เช็คเครื่องมือ"
    3. ช่างกดปุ่ม เปิดหน้า LIFF ให้ติ๊ก Checklist และแนบรูปรวมเครื่องมือ
    4. ข้อมูลบันทึกลง Google Sheets หากใครไม่ทำ แอดมินจะได้รับการแจ้งเตือน
*   **เทคโนโลยี:** LINE Flex Message, LINE LIFF, Google Apps Script
*   **ข้อมูลที่ต้องเก็บ:** Audit Date, Employee ID, Tool Status, Audit Photo URL

#### 3.2.4 ออกร่างใบสั่งซื้ออัตโนมัติ (Automated Draft PO)
*   **คำอธิบาย:** เมื่อสินค้าในคลังหลักลดลงถึงจุดสั่งซื้อซ้ำ (Reorder Point) ระบบจะสร้างไฟล์ PDF ใบสั่งซื้อ (Purchase Order) ส่งให้ผู้มีอำนาจอนุมัติทาง LINE ทันที
*   **User Story:** ในฐานะฝ่ายจัดซื้อ ฉันต้องการให้ระบบเตรียมใบสั่งซื้อให้เมื่อของใกล้หมด เพื่อความรวดเร็วในการสั่งของ
*   **Flow การทำงาน:**
    1. ทุกครั้งที่มีการตัดสต๊อก GAS จะตรวจสอบว่า Quantity <= Reorder Point หรือไม่
    2. หากใช่ GAS จะดึงข้อมูล Vendor ที่ผูกไว้กับสินค้านั้น
    3. สร้างไฟล์ PDF จาก Google Docs Template
    4. ส่ง PDF เข้า LINE แอดมิน พร้อมปุ่ม "อนุมัติสั่งซื้อ"
*   **เทคโนโลยี:** Google Docs API (Templating), Google Apps Script, LINE Flex Message
*   **ข้อมูลที่ต้องเก็บ:** Reorder Point, Vendor Info, Item Cost

### 💰 3.3 กลุ่มฟีเจอร์การเงินและการขาย (Finance & Sales)
**เป้าหมาย:** ปิดการขายไว รับเงินทันที

#### 3.3.1 PromptPay Automation
*   **คำอธิบาย:** ทันทีที่ช่างกด "ปิดงาน" ระบบจะสร้าง QR Code พร้อมเพย์ที่ระบุยอดเงินเป๊ะๆ ส่งเข้า LINE ลูกค้า (หรือให้ช่างเปิดให้ลูกค้าสแกน) เพื่อความสะดวกและลดข้อผิดพลาดในการโอนเงิน
*   **User Story:** ในฐานะลูกค้า ฉันต้องการสแกนจ่ายเงินได้เลยโดยไม่ต้องพิมพ์ตัวเลขยอดเงินเอง
*   **Flow การทำงาน:**
    1. ช่างกดปิดงาน ระบบดึงยอดเงินคงเหลือที่ต้องชำระจากใบงาน
    2. GAS สร้าง PromptPay Payload string (ประกอบด้วยเบอร์พร้อมเพย์ร้านและยอดเงิน)
    3. แปลง Payload เป็นรูป QR Code ผ่าน API สร้าง QR
    4. ส่งรูป QR Code เข้า LINE ลูกค้า หรือแสดงบนหน้าจอ LIFF ของช่าง
*   **เทคโนโลยี:** PromptPay QR Standard, QR Code Generator API
*   **ข้อมูลที่ต้องเก็บ:** Biller ID (Tax ID/Phone), Invoice Amount

#### 3.3.2 ระบบตรวจสลิปและออกใบเสร็จ (Slip Verification & Auto-Receipt)
*   **คำอธิบาย:** เมื่อลูกค้าโอนเงินและส่งสลิปเข้า LINE ระบบจะตรวจสอบความถูกต้องของสลิป (ยอดเงิน, บัญชีผู้รับ) ผ่าน API หากถูกต้อง จะสร้างใบเสร็จรับเงิน PDF ส่งกลับให้ลูกค้าทันที
*   **User Story:** ในฐานะแอดมินการเงิน ฉันต้องการให้ระบบตรวจสลิปปลอมและออกใบเสร็จให้ลูกค้าอัตโนมัติ เพื่อลดงานเอกสาร
*   **Flow การทำงาน:**
    1. ลูกค้าส่งรูปสลิปเข้า LINE
    2. GAS ส่งรูปไปให้ Slip Verification API (เช่น EasySlip หรือ Bank API) อ่าน QR บนสลิป
    3. ตรวจสอบยอดเงิน (Amount) และบัญชีผู้รับ (Receiver) ว่าตรงกับใบงานหรือไม่
    4. หากตรง GAS สร้างใบเสร็จ PDF จากเทมเพลต และส่งให้ลูกค้า
*   **เทคโนโลยี:** Slip Verification API (3rd Party), Google Docs API, Google Apps Script
*   **ข้อมูลที่ต้องเก็บ:** Slip Payload, Transaction Ref, Receipt PDF URL

#### 3.3.3 เช็คราคากลาง ICT (Price Benchmarking)
*   **คำอธิบาย:** ระบบดึงข้อมูลราคากลางของอุปกรณ์คอมพิวเตอร์และกล้องวงจรปิด (อ้างอิงประกาศกระทรวงดิจิทัลฯ ล่าสุด เช่น ฉบับเดือนธันวาคม 2568) มาแสดงเทียบให้ดูขณะทำใบเสนอราคา เพื่อรองรับงานราชการ
*   **User Story:** ในฐานะฝ่ายขาย ฉันต้องการรู้ราคากลาง ICT ทันทีตอนทำใบเสนอราคา เพื่อไม่ให้เสนอราคาเกินกำหนดของราชการ
*   **Flow การทำงาน:**
    1. จัดทำฐานข้อมูลราคากลาง ICT ใน Google Sheets (เช่น NVR 16 ช่อง ราคา 57,000 บาท)
    2. เมื่อฝ่ายขายเลือกสินค้าในหน้าสร้างใบเสนอราคา ระบบจะดึงราคากลางมาแสดงเทียบกับราคาขายของร้าน
*   **เทคโนโลยี:** Google Sheets, LINE LIFF
*   **ข้อมูลที่ต้องเก็บ:** ICT Standard Price List

#### 3.3.4 สร้างใบเสนอราคาด่วน (Automated Quotation)
*   **คำอธิบาย:** ช่างหรือเซลส์สามารถกดปุ่มเลือกแพ็กเกจสินค้าผ่าน LINE LIFF แล้วระบบจะสร้างใบเสนอราคา (Quotation) เป็น PDF ส่งให้ลูกค้าดูได้ทันทีจากหน้างาน
*   **User Story:** ในฐานะเซลส์ ฉันต้องการส่งใบเสนอราคาให้ลูกค้าดูทันทีที่คุยจบ เพื่อเพิ่มโอกาสปิดการขาย
*   **Flow การทำงาน:**
    1. เซลส์เปิดหน้า LIFF เลือกแพ็กเกจ (เช่น เซ็ตกล้อง 4 ตัว) และใส่ชื่อลูกค้า
    2. GAS รับข้อมูล นำไปแทนที่ตัวแปรใน Google Docs Template
    3. แปลงเป็น PDF และส่งลิงก์หรือไฟล์เข้า LINE เซลส์
*   **เทคโนโลยี:** LINE LIFF, Google Docs API, Google Drive
*   **ข้อมูลที่ต้องเก็บ:** Customer Name, Selected Package, Quotation PDF URL

### 📈 3.4 กลุ่มฟีเจอร์การตลาดและลูกค้าสัมพันธ์ (Marketing & CRM)

#### 3.4.1 นักการตลาด AI (AI Content Marketing)

#### 3.4.2 ระบบดูแลหลังการขาย (Predictive Maintenance)
*   **คำอธิบาย:** AI วิเคราะห์ประวัติลูกค้า หากใกล้หมดประกัน 1 ปี หรือเข้าสู่ช่วงฤดูฝน ระบบจะส่งข้อความ LINE ทักไปหาลูกค้าเพื่อเสนอโปรโมชั่นตรวจเช็คระบบไฟหรือล้างทำความสะอาดกล้อง
*   **User Story:** ในฐานะฝ่ายการตลาด ฉันต้องการสร้างรายได้จากลูกค้าเก่าด้วยบริการดูแลหลังการขาย
*   **Flow การทำงาน:**
    1. GAS รัน Trigger ตรวจสอบวันที่ติดตั้ง (Installation Date) ของลูกค้าทั้งหมด
    2. หากครบ 11 เดือน หรืออยู่ในเดือนพฤษภาคม (เข้าหน้าฝน)
    3. ส่งข้อความ Personalized Message หาลูกค้าผ่าน LINE OA (เช่น "สวัสดีครับคุณ A กล้องวงจรปิดใกล้ครบ 1 ปีแล้ว รับโปรโมชั่นตรวจเช็คระบบไหมครับ")
*   **เทคโนโลยี:** Google Apps Script (Time-driven triggers), LINE Messaging API
*   **ข้อมูลที่ต้องเก็บ:** Installation Date, Customer LINE User ID, Maintenance History

#### 3.4.3 แผนที่ลูกค้า (Heatmap Analytics)
*   **คำอธิบาย:** นำพิกัด GPS ของลูกค้าทั้งหมดมาแสดงเป็นแผนที่ความร้อน (Heatmap) บนเว็บดาชบอร์ด เพื่อให้เห็นพื้นที่ที่มีฐานลูกค้าหนาแน่น ใช้ประกอบการตัดสินใจยิงแอดโฆษณาหรือตั้งสาขาใหม่
*   **User Story:** ในฐานะผู้บริหาร ฉันต้องการเห็นภาพรวมว่าลูกค้าในจังหวัดร้อยเอ็ดกระจุกตัวอยู่ที่อำเภอไหนมากที่สุด
*   **Flow การทำงาน:**
    1. ดึงข้อมูล Latitude/Longitude ของลูกค้าทั้งหมดจาก Google Sheets
    2. นำไปพล็อตบน Google Maps ผ่าน Google Maps JavaScript API (Heatmap Layer)
    3. แสดงผลบนหน้า Web App (Google Apps Script HTML Service)
*   **เทคโนโลยี:** Google Maps JavaScript API, GAS HTML Service
*   **ข้อมูลที่ต้องเก็บ:** Customer Coordinates (Lat/Long)

## 4. Flow ทะลวงกำแพง — 1 งาน เชื่อม 4 ห้อง (Cross-Room Automation Flow)

ระบบนี้ออกแบบมาเพื่อเชื่อมโยงการทำงานของทุกแผนกเข้าด้วยกันอย่างไร้รอยต่อ โดยใช้บอทเป็นตัวกลางในการสื่อสารและส่งต่องานอัตโนมัติ เพื่อลดความผิดพลาดและเพิ่มความโปร่งใส

**ตัวอย่างสถานการณ์:** ลูกค้าตกลงติดตั้งกล้องวงจรปิด 1 งาน

| แผนก / ห้องแชท | กระบวนการทำงาน |
| :--- | :--- |
| **💬 ห้องที่ 1 — ห้องการขาย (Sales Room)** | • เซลส์พิมพ์คำสั่งเปิดงานพร้อมระบุรายละเอียดลูกค้า<br>• บอทอ่านข้อมูลและเปิด Job ID ใหม่ใน Google Sheet (DB_JOBS) พร้อมสร้างโฟลเดอร์ใน Drive<br>• **[แจ้งเตือนข้ามห้อง]** บอทส่งข้อความไปที่ "ห้องช่าง" พร้อมสรุปรายละเอียดงาน แผนที่ และเบอร์ติดต่อลูกค้า |
| **🛠️ ห้องที่ 2 — ห้องช่าง (Field Tech Room)** | • ช่างสแกน QR Code เพื่อเริ่มงาน/ปิดงาน และส่งรูป Before/After ลงแชท<br>• บอทจัดเก็บรูปเข้า Drive จัดเรียงเข้าโฟลเดอร์ และอัปเดตสถานะงานเป็น "ติดตั้งเสร็จสิ้น"<br>• **[แจ้งเตือนข้ามห้อง]** บอทส่งข้อความไปที่ "ห้องบัญชีการเงิน" แจ้งว่า "งาน Job-001 ติดตั้งเสร็จแล้ว กรุณาออกอินวอยซ์"<br>• **[ตัดสต๊อก]** บอทตรวจสอบรายการอุปกรณ์ที่ใช้และตัดยอดในฐานข้อมูลสต๊อก (DB_INVENTORY) อัตโนมัติ |
| **💰 ห้องที่ 3 — ห้องบัญชีการเงิน (Finance Room)** | • แอดมินบัญชีเห็นแจ้งเตือน กดยืนยันเพื่อออกอินวอยซ์หรือส่ง QR Code รับเงินให้ลูกค้า<br>• เมื่อลูกค้าโอนเงิน แอดมินส่งสลิปลงห้อง<br>• บอทใช้ OCR อ่านสลิปและตรวจสอบยอดเงินว่าตรงกับใบเสนอราคาหรือไม่<br>• บอทอัปเดตสถานะเป็น "ชำระเงินแล้ว" และส่งใบเสร็จให้ลูกค้า<br>• **[แจ้งเตือนข้ามห้อง]** บอทส่งข้อความไปที่ "ห้องผู้บริหาร" แจ้งว่า "รับยอดงาน Job-001 แล้ว 25,000 บาท" |
| **📦 ห้องที่ 4 — ห้องสั่งสินค้า (Procurement Room)** | • จากการตัดสต๊อกในห้องช่าง หากพบว่าสินค้าลดลงต่ำกว่าจุดสั่งซื้อ (Min Threshold)<br>• บอทจะส่งแจ้งเตือน: "สินค้าใกล้หมด: สายแลน CAT6 เหลือ 1 ม้วน ต้องการร่างใบสั่งซื้อไหม?"<br>• หากแอดมินพิมพ์ "ตกลง" บอทจะสร้างร่างใบสั่งซื้อ (PO) เป็นไฟล์ PDF พร้อมส่งอีเมลไปยังผู้ผลิตได้ทันที |

**จุดเด่นของ Flow นี้:**
*   **ไม่ต้องตามงาน:** ทุกคนสามารถทำงานในพื้นที่ของตัวเองได้ โดยมีบอททำหน้าที่เป็น "เด็กเดินเอกสาร" คอยส่งข่าวและประสานงานข้ามแผนก
*   **ข้อมูลโปร่งใส:** ทุกขั้นตอนถูกบันทึกลงใน Google Sheet แบบ Real-time ทำให้ผู้บริหารสามารถดูภาพรวมผ่าน Dashboard ได้ตลอดเวลา
*   **ลด Human Error:** ป้องกันปัญหาการลืมจดบันทึกของช่าง หรือการลืมเก็บเงิน/ออกใบเสร็จของฝ่ายบัญชี

## 5. ข้อเสนอฟีเจอร์เพิ่มเติม (Additional Feature Proposals)

เพื่อเพิ่มขีดความสามารถของระบบให้สมบูรณ์ยิ่งขึ้น ขอเสนอแนวทางพัฒนาเพิ่มเติมดังนี้:

1.  **Customer Self-Service Portal (ระบบแจ้งซ่อมด้วยตนเอง):**
    *   สร้าง LINE Rich Menu ให้ลูกค้าสามารถกด "แจ้งซ่อม" ถ่ายรูปอาการเสีย และเลือกวันเวลานัดหมายได้เอง ข้อมูลจะวิ่งเข้าคิวงานของช่างโดยตรง ลดภาระแอดมิน
2.  **IoT Integration สำหรับอุปกรณ์สำคัญ:**
    *   สำหรับลูกค้าระดับองค์กร (B2B) สามารถเชื่อมต่อระบบ Monitoring ของ NVR เข้ากับ Super App หากกล้องดับหรือฮาร์ดดิสก์พัง ระบบจะสร้างใบงาน (Auto-Ticketing) แจ้งเตือนช่างทันทีก่อนที่ลูกค้าจะรู้ตัว
3.  **Gamification & Technician Performance (ระบบสะสมแต้มช่าง):**
    *   ให้คะแนนช่างจากการปิดงานตรงเวลา การไม่ลืมเครื่องมือ และคะแนนรีวิวจากลูกค้า นำมาแสดงเป็น Leaderboard ประจำเดือนเพื่อรับโบนัส สร้างแรงจูงใจในการทำงาน
4.  **E-Signature สำหรับเอกสารรับมอบงาน:**
    *   เมื่อช่างติดตั้งเสร็จ ให้ลูกค้าเซ็นรับมอบงานด้วยนิ้วบนหน้าจอโทรศัพท์ของช่างผ่านระบบ LIFF ลายเซ็นจะถูกแนบลงในเอกสาร PDF อัตโนมัติ เพิ่มความน่าเชื่อถือทางกฎหมาย

## 6. Roadmap การพัฒนา (Development Roadmap)

การพัฒนาจะแบ่งออกเป็น 3 ระยะ (Phases) เพื่อให้ระบบสามารถนำไปใช้งานจริงได้เร็วที่สุด (MVP) และค่อยๆ เพิ่มความซับซ้อน

*   **Phase 1: Core Operations & Zero-Data Entry (เดือนที่ 1-2)**
    *   ระบบสแกน QR Code 12 สถานะ
    *   ระบบคัดแยกรูปภาพอัจฉริยะและ Vision Agent ลง Google Drive
    *   ระบบดึงพิกัดอัตโนมัติ (Geofencing)
    *   สร้างใบเสนอราคาด่วน (Automated Quotation)
*   **Phase 2: Inventory & Finance Automation (เดือนที่ 3-4)**
    *   ระบบสต๊อก 3 ชั้น
    *   แจ้งเตือนลืมเครื่องมือ และตรวจเช็คเครื่องมือรายสัปดาห์
    *   PromptPay Automation และระบบตรวจสลิป
    *   เช็คราคากลาง ICT
*   **Phase 3: AI & Advanced CRM (เดือนที่ 5-6)**
    *   สั่งงานด้วยเสียง (Voice-to-Workflow)
    *   ระบบพยากรณ์ของขาด (Predictive Stocking)
    *   ระบบดูแลหลังการขาย (Predictive Maintenance) และ Heatmap
    *   ระบบ Cross-Room Automation Flow แบบเต็มรูปแบบ

## 7. ข้อจำกัดและความเสี่ยงทางเทคนิค (Technical Limitations & Risks)

เนื่องจากระบบพึ่งพา Google Apps Script (GAS) เป็นแกนหลัก จึงมีข้อจำกัดด้าน Quota ของ Google ที่ต้องวางแผนรับมือ:

1.  **Execution Time Limit:** GAS มีขีดจำกัดเวลาทำงาน 6 นาทีต่อการรัน 1 ครั้ง การอัปโหลดไฟล์ขนาดใหญ่หรือการประมวลผลรูปภาพจำนวนมากอาจทำให้เกิด Time-out ได้ *แนวทางแก้ไข: ใช้ระบบ Queue หรือแบ่งการทำงานเป็นชุดย่อย (Chunking)* [1]
2.  **Email & API Limits:** บัญชี Google Workspace ทั่วไปจำกัดการส่งอีเมลที่ 1,500 ฉบับ/วัน และ URL Fetch calls (การเรียก API ภายนอก) ที่ 100,000 ครั้ง/วัน *แนวทางแก้ไข: ติดตามการใช้งาน API อย่างใกล้ชิด และใช้ Caching หากเป็นไปได้* [1]
3.  **Concurrent Executions:** GAS รองรับการทำงานพร้อมกันสูงสุด 30 executions ต่อผู้ใช้ หากช่างหลายคนส่งรูปภาพและเรียกใช้งานพร้อมกันในเสี้ยววินาทีอาจเกิดคอขวด *แนวทางแก้ไข: ใช้เทคนิค Exponential Backoff ในการเรียกซ้ำเมื่อเกิด Error* [1]
4.  **ความแม่นยำของ GPS:** การดึงพิกัดจากรูปภาพขึ้นอยู่กับการตั้งค่าโทรศัพท์ของช่าง หากช่างปิด Location Service รูปภาพจะไม่มี EXIF GPS *แนวทางแก้ไข: บังคับให้ส่ง Location Message ผ่าน LINE ควบคู่กัน*
5.  **ต้นทุน API ภายนอก:** การใช้ OpenAI API สำหรับเสียงและรูปภาพ, รวมถึง Slip Verification API มีค่าใช้จ่ายเป็นรายครั้ง (Pay-per-use) จำเป็นต้องประเมินปริมาณการใช้งานเพื่อควบคุมต้นทุน
